# Инфраструктура вычислений в биоинформатике.

Репозиторий посвящён курсу на Факультете Биоинженерии и Биоинформатики МГУ.

Курс посвящён современным подходам к организации вычислительной инфраструктуры в 
биоинформатике и смежных областях анализа данных. Рассматривается полный стек 
создания приложения: от разработки кода и взаимодействия с внешними API до 
развёртывания контейнеризованных вычислений, автоматизации пайплайнов, 
мониторинга и управления экспериментами машинного обучения. Особое 
внимание уделяется воспроизводимости, масштабируемости и 
безопасности вычислений в научной среде.

В рамках курса изучаются системы контроля версий и CI/CD, принципы построения 
и использования API, основы компьютерных сетей и криптографии, контейнеризация 
вычислений, работа с вычислительными кластерами, автоматизация задач, разработка 
пользовательских сервисов визуализации, а также оркестрация и мониторинг пайплайнов. 
Завершающая часть курса посвящена управлению экспериментами и моделями машинного 
обучения. Курс ориентирован на студентов, планирующих разрабатывать и поддерживать 
воспроизводимые вычислительные системы и пайплайны в биоинформатике.

## План курса:

В курсе планируется 4 блока.

  1. Системы контроля версий и работа с API.
  2. Компьютерные сети и криптография.
  3. Контейнеризация, работа на HPC, автоматизация.
  4. Пайплайны обработки данных, их мониторинг и презентация результатов.

## Стек курса:

Ожидается, что к концу курса все будут ознакомлены с:
* Системами контроля версий и веб-хранилищами git, Github, 
* CI/CD с помощью Github Actions;
* Пакетами python для работы с интернетом: `BeautifulSoup`, `requests`, `scrapy`, `selenium`;
* Принципами REST, gRPC, GraphQL API;
* Фреймворками для построения своего API: `FastAPI`, `pydantic`;
* Telegram Bot API;
* Стеком сетевых протоколов TCP/IP;
* Утилитами командной строки для работы с различными уровнями сетевых протоколов;
* Простейшими алгоритмами криптографии (RSA, Диффи-Хельман);
* Принципами контейнеризации в Docker, Apptainer;
* Инструментами для работы на вычислительном кластере (SLURM, `module`);
* Простейшим планировщиком задач `cron`;
* Конвейерами обработки данных, сконструированных в `airflow`;
* Мониторинг состояния сервисов с помощью Grafana;
* Управление пайплайнов машинного обучения с MLflow.
